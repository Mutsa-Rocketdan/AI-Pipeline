## 2. 정성적 분석

### 2.1 데이터 형식

- **포맷**: 한 줄당 한 발화. `\<HH:MM:SS> 화자ID: 발화 내용` 형태.
- **인코딩**: UTF-8.
- **출처**: STT(음성 인식) 변환 결과로 추정되며, 구어체·오타·반복이 일부 포함됨.

**예시 (실제 라인):**
```
<09:11:17> b54f46b0: 여러분 오늘 수업 진행하도록 하겠습니다. 저희가 이제 오늘 1차 잡바 마지막 날입니다.
<09:12:04> b54f46b0: 아이오 패키지가 있고 그다음에 밑에 보시면은 자바 NIO라는 패키지가 있죠.
```

### 2.2 화자

- **화자 ID**: 8자리 16진수 (예: `b54f46b0`, `dffb71cf`, `f1db1019`, `f3a139ae`).
- 한 파일 내에서 주로 1~2개 ID가 등장하며, 강사/발표자 구간으로 해석 가능.

### 2.3 커리큘럼 대비 실제 내용 (정성)

| 주차 | 커리큘럼 상 주제 | 스크립트에서 드러나는 실제 강의 내용 |
|------|------------------|----------------------------------------|
| 1주 | 객체지향·프론트엔드(HTML/CSS/JS) | 02-02: **Java I/O, NIO, NIO2** 등 백엔드/자바. 03~06: **MySQL, Workbench, SELECT, GROUP BY, 데이터 타입** 등 DB/SQL 중심. |
| 2주 | React, 테일윈드 | **JOIN, 서브쿼리, CTE, CREATE TABLE, 제약 조건** 등 SQL 심화. |
| 3주 | React Query, 백엔드(HTTP) | **뷰, 프로시저, 파티션, 인덱스, 사용자 권한, 트리거, 덤프/복구, 데이터 모델링** 등 DB 운영·설계. |

- **정성적 특징**: 공식 커리큘럼(객체지향·프론트엔드·React)과 달리, 스크립트 상으로는 **Java 기초, MySQL/SQL, DB 설계·운영** 비중이 큼. 퀴즈·학습 가이드 생성 시 **실제 스크립트 내용**을 기준으로 하는 것이 적절함.

### 2.4 품질·노이즈

- **타임스탬프**: 대체로 연속적이나, 동일 초가 반복되는 구간 있음.
- **오타/구어**: 예) "잡바"→자바, "효울씨"→효율, "N아I어"→NIO, "3레드"→스레드 등. 전처리에서 타임스탬프·화자ID만 제거하고, 의미 해석은 임베딩·LLM에 위임하는 구조가 적합함.
- **발화 단위**: 문장이 짧게 잘려 있는 경우가 많아, 세그먼트·청킹 후 사용하는 현재 파이프라인이 적절함.

## 3. 프로젝트 활용 관점

| 항목 | 내용 |
|------|------|
| **전처리** | 타임스탬프·화자ID 제거, 시간 gap 기준 세그먼트, 토큰 기준 청킹(600 토큰, 100 오버랩) 적용 시 약 1,500개 내외 청크 생성 가능. |
